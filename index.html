자율 코딩 에이전트를 위한 GitHub 중심 컨베이어 벨트

GitHub = durable state machine Issues = 작업 큐 PRs = 리뷰 큐 Labels = 현재 상태 CI = 품질 게이트

별도 DB 없이 GitHub 자체를 상태 저장소로 씁니다 — 위 네 가지가 각각 그 역할을 맡습니다.

에이전트(claude / codex)는 헤드리스 모드의 stateless 워커 — 격리된 git worktree에서 한 번에 하나의 작업만 수행합니다.
두 리뷰어가 합의에 이르지 못하면 중재자(Arbiter)가 최종 결정(SHIP / REDO)을 내려 막다른 길을 없앱니다 (agent:blocked만 드문 예외).

대기 중
—
배치 뷰 — Planner 큐(이슈 10개)를 벨트가 어떻게 소비하는가 · 위 벨트 = 이슈 1개의 상세 여정 / 아래 = 큐 전체
대기10 처리 중0 완료0 막힘0 동시성 = 1 · coder = 최하위 우선순위
대기 큐agent:ready · FIFO
처리 중 (1)coder → review → merge
완료 / 막힘agent:done · blocked
단일 레인: 코더는 최하위 우선순위라, 하나의 PR이 머지(또는 agent:blocked)될 때까지 다음 agent:ready 이슈를 집지 않습니다 — 큐는 한 번에 하나씩 빠집니다. 동시성은 옵트인(워크트리 다중화).
